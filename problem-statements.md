TITLE-	BUILDATLAS GENAI – SMART CONSTRUCTION INTELLIGENCE PLATFORM
Description	
The construction industry is one of the key drivers of infrastructure and economic development, with thousands of residential, commercial, and industrial projects executed every year. However, many engineers and project planners face challenges in accurately estimating project costs, planning schedules, and allocating resources efficiently. Incorrect estimations and poor planning often lead to budget overruns, project delays, and inefficient use of materials and workforce. Many small and medium construction firms rely on manual calculations, spreadsheets, and fragmented tools, which increases the chances of errors and reduces overall productivity. Additionally, analyzing project requirements and interpreting planning data requires expertise and time, making the process difficult for less experienced engineers. With the advancement of Generative AI and Large Language Models (LLMs), there is an opportunity to build an intelligent and scalable construction planning system that can analyze project parameters and generate accurate cost estimates, optimized schedules, and resource plans to improve project efficiency and decision-making.
The objective of this use case is to develop a Generative AI-powered prototype that can act as a construction planning copilot for engineers, architects, and project managers. The system should be able to analyze project inputs such as project type, dimensions, materials, and requirements to generate cost estimates, recommend resource allocation, and create optimized construction schedules. It should also provide project insights and conversational planning support through an AI assistant. The goal is to create a scalable solution that can help construction professionals make informed planning decisions and execute projects more efficiently.

Expected Solutions	The expected solution is a functional web-based prototype that allows users to enter project details such as building type, size, and materials, and then uses Generative AI to analyze the inputs and generate estimated project costs, resource requirements, and construction schedules. The system should provide structured planning outputs and recommendations to improve project efficiency and accuracy. Additionally, the platform should include an AI-powered chatbot that allows users to ask construction-related questions and receive intelligent planning guidance. The focus should be on building a minimum viable product that demonstrates the core functionality.
Technologies & Tools	
Participants are encouraged to use modern and scalable technologies to build the solution. The backend can be developed using Python with FastAPI or Node.js, while the frontend can be built using Streamlit for rapid prototyping or React for advanced user interfaces. Generative AI models such as IBM Watson AI, Gemini, GPT, Llama, or Mistral can be used to generate cost estimates and planning guidance. Data processing can be implemented using libraries such as Pandas and NumPy. Teams can optionally use Retrieval-Augmented Generation (RAG) and vector databases such as FAISS or Pinecone to improve accuracy. Deployment can be done using cloud platforms such as Local Hosting, AWS, IBM Cloud, Firebase, or Streamlit Cloud.

TITLE	CLAIMSHIELD ML – INTELLIGENT INSURANCE FRAUD DETECTION PLATFORM
Description	
The insurance industry processes millions of claims every year, making fraud detection a critical challenge for insurance providers. Fraudulent claims result in significant financial losses and increase operational costs for insurance companies. Many traditional fraud detection methods rely on manual review processes and rule-based systems, which are time-consuming, inefficient, and often unable to detect complex fraud patterns. Additionally, the increasing volume and complexity of insurance data make it difficult for investigators to identify suspicious claims accurately. Without intelligent systems, fraudulent activities may go undetected, affecting company profitability and increasing premiums for genuine customers. With the advancement of Machine Learning, there is an opportunity to build an intelligent fraud detection system that can analyze claim data, identify suspicious patterns, and help insurance companies detect and prevent fraudulent claims more effectively.

The objective of this use case is to develop a Machine Learning-powered prototype that can act as an intelligent fraud detection system for insurance providers. The system should be able to analyze insurance claim details, identify patterns, and classify claims as legitimate or potentially fraudulent. It should also generate risk scores and provide insights to support fraud investigation. The goal is to create a scalable solution that improves fraud detection accuracy and helps insurance companies reduce financial losses and improve operational efficiency.

Expected Solutions	The expected solution is a functional web-based prototype that allows users to input insurance claim data, and then uses Machine Learning algorithms to analyze the data and detect potential fraud. The system should classify claims based on risk levels and provide risk scores and supporting insights. Additionally, the platform should provide decision support by highlighting suspicious patterns and enabling investigators to review high-risk claims efficiently. The focus should be on building a minimum viable product that demonstrates accurate fraud detection and classification capabilities.
Technologies & Tools	
Participants are encouraged to use modern and scalable technologies to build the solution. The backend can be developed using Python, while the frontend can be built using Streamlit or React for interactive user interfaces. Machine Learning libraries such as Pandas and NumPy can be used for data processing, and Scikit-learn can be used to implement models such as Random Forest, Decision Trees, or XGBoost for fraud detection. Data visualization libraries such as Matplotlib or Seaborn can be used to display insights and model performance metrics. Deployment can be done using cloud platforms such as Local Hosting, AWS, Firebase, or Streamlit Cloud.
